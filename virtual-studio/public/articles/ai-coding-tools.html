01

先建框架：别把发动机当汽车

网上讨论 AI 编程工具时，最常见的混乱是：把底层模型和上层工具放在一起比较，就像同时讨论"丰田的发动机"和"特斯拉这台车"谁更好开——两个维度，根本不是一个问题。

理解这个圈子，先记住三层架构：

🧠
底层模型
提供智能能力，不直接面向用户使用
Claude Opus 4.6 / GPT-5.4 / Gemini 3.1 Pro
⚙️
编程 Agent / 工具
调用模型执行工程任务，有工作流和上下文管理
Claude Code / Codex / Gemini CLI
🖥️
IDE / 平台
提供编写代码的完整环境，AI 嵌入其中
Cursor / Antigravity / Copilot
换位思考：你买车不需要懂发动机原理，但你得知道这台车用的是什么发动机——它直接影响性能上限。同理，选 AI 编程工具时，背后调用什么模型，决定了这个工具的天花板。
02

底层模型扫盲（引擎层）

这些模型你大概率不会直接用，但它们是所有工具的能力来源。你选的工具背后调用哪个模型，直接影响代码质量、推理深度和上下文长度。

模型 厂商 核心特点 编程能力 上下文
Claude Opus 4.6 Anthropic 顶级推理，长任务最稳 SWE-bench ~80.8%，业界第一 1M tokens
Claude Sonnet 4.6 Anthropic 速度与质量平衡，日常主力 极强，大多数场景够用 200K tokens
GPT-5.4 OpenAI 多模态综合表现强，生态最大 全面，多模态理解突出 128K tokens
codex-1 (o3架构) OpenAI 专为编程优化的推理模型 复杂工程任务推理深 200K tokens
Gemini 3.1 Pro Google 超长上下文，多模态 强，但略逊前三 1M tokens
DeepSeek V3 / R1 深度求索 开源，极致性价比 相当不错，远超价格预期 128K tokens
Llama 4 Scout/Maverick Meta 开源，可完全本地部署 中等，本地跑有上限 128K tokens
SWE-bench 是什么？ 一个标准化测试集，让模型在真实 GitHub Issue 上独立修复 Bug，满分100%。80% 以上意味着模型能独立解决大多数真实工程问题，这是目前最有参考价值的编程能力基准。
一个常见认知误区：模型强 ≠ 工具好用。一个顶级模型配上糟糕的上下文管理、差劲的工作流，使用体验照样很烂。工具层的工程质量同样关键。
03

终端 Agent（命令行派）

这类工具跑在终端里，没有 GUI，直接操作你的代码库文件系统、运行命令、调用 Git。看起来硬核，但却是目前自动化能力最强的一类工具——因为它不需要模拟鼠标点击，可以直接做所有事情。

适合人群：能接受命令行的开发者；需要跑长任务、批量重构的场景。

Claude Code
Anthropic · 2025年3月发布
CLI Agent
Pro $17/月 · Max $100/月

目前综合评价最强的 CLI Agent。背靠 Opus 4.6，自发布8个月内跃升为使用率第一的 AI 编程工具（SuperClaude 社区数据）。核心能力：多文件并行编辑、自动运行测试、深度 Git 集成、支持 Multi-Agent 并行子任务。你给它一个需求，它会自己读代码库、写代码、跑测试、修 Bug，一条龙完成。

✦ 优势
  • 推理能力最强，复杂重构表现顶级
  • 上下文管理最成熟
  • Multi-Agent 团队协作
  • 与 Claude.ai 订阅共享额度
✦ 不足
  • 纯命令行，无可视化
  • Max 套餐 $100/月，重度使用成本高
  • 网络要求稳定
OpenAI Codex（重构版）
OpenAI · 2026年3月重磅回归
CLI 桌面 App NEW
捆绑 ChatGPT Plus $20/月

重要提醒： 这不是2021年那个 Codex（那版已停用）。新版基于 codex-1 模型（o3 架构专门针对工程任务调优），有开源 CLI 和 macOS 桌面客户端两种形态。核心差异化：沙箱环境执行，代码在隔离容器里跑，安全性更高。ChatGPT Plus/Pro 订阅用户可直接访问。

✦ 优势
  • 已订阅 ChatGPT 则无额外费用
  • 沙箱执行，不污染本地环境
  • macOS 桌面版体验友好
  • 开源 CLI 可自定义
✦ 不足
  • 与 Claude Code 相比推理深度略逊
  • Windows 支持尚不完善
  • 重度任务额度消耗快
Gemini CLI
Google · 2025年下半年发布
CLI 免费额度大
免费 1000次/天

最具性价比的入门 CLI Agent。免费额度慷慨（每日1000次请求），调用 Gemini 3.1 Pro，支持 1M tokens 超长上下文，这意味着可以一次性读入整个大型代码库。对预算敏感的开发者是很好的起点。

注意：免费额度的多数请求实际走 Gemini Flash 轻量模型，复杂任务效果会下降；Pro 级别调用需要升级付费。

✦ 优势
  • 超高免费额度，0成本上手
  • 1M 上下文，读大仓库得心应手
  • Google 生态深度集成
✦ 不足
  • 免费版实际走轻量模型
  • 成熟度不如 Claude Code
OpenCode
开源社区
开源 任意模型
免费（自付 API 费用）

类似 Claude Code 的 CLI 体验，但完全开源，支持接入任意模型（Claude、GPT、DeepSeek、Gemini 等）。你自己带 API Key，按量计费，成本透明可控。适合：有技术能力、想省钱、或有隐私顾虑（自部署模型）的开发者。

Aider
开源 · 老牌稳定
开源 Git 集成
免费（自付 API 费用）

开源 CLI Agent 的先驱，与 Git 深度集成，每次修改自动提交，方便回滚。支持接入 Claude、GPT、DeepSeek 等主流模型。社区成熟，文档齐全，是学习 CLI Agent 工作原理的好选择。

04

AI-first IDE（图形化派）

不喜欢命令行？这类工具有完整的图形编辑器，AI 深度嵌入编写流程，你在代码里写注释它就开始补全，你选中一段代码按快捷键它就开始重构。学习曲线低，日常用起来最顺手。

Cursor
Anysphere · 目前最成熟的 AI IDE
AI IDE VS Code 派系
免费有限 · Pro $20/月 · Business $40/月

目前用户基数最大、生态最成熟的 AI IDE。基于 VS Code 改造，你原来的插件、快捷键、配置全部能用。核心功能：Composer（多文件联动编辑，描述需求自动修改多个文件）、Tab（下一步预测式自动补全，远超传统补全体验）、模型自由切换（Claude Sonnet / GPT-5.4 / Gemini 等）。

可以理解为：把 Claude Code 的能力装进一个你熟悉的 VS Code 界面里。

✦ 优势
  • VS Code 用户零迁移成本
  • 多文件 Composer 体验业界最佳
  • 模型可灵活切换
  • 社区最大，教程资源丰富
✦ 不足
  • 重度使用额度消耗快
  • Pro 版 $20/月，重度用户需 Business
  • 本身不含模型，质量依赖所选模型
Google Antigravity
Google · 2025年11月发布
AI IDE NEW · Beta
Beta 免费 · 正式定价 TBD

Google 对 Cursor 的直接回应，定位原生 Multi-Agent IDE。最大亮点：Mission Control 模式——可以同时运行多个 AI Agent 并行处理不同任务（前端 Agent 写 UI、后端 Agent 写 API、测试 Agent 跑测试），最终合并结果。内置浏览器，可以直接在 IDE 里调试网页。

支持模型：Gemini 3 Ultra / Claude Sonnet 4.5 / GPT 系列。目前仍在 Beta，稳定性不如 Cursor，但代表了 AI IDE 的下一阶段方向。

✦ 优势
  • 多 Agent 并行是独家特性
  • 内置浏览器调试体验
  • Google 生态深度集成
  • Beta 阶段免费
✦ 不足
  • Beta 阶段，稳定性尚待验证
  • 正式定价未公布
  • 学习曲线比 Cursor 陡
Windsurf
Codeium · 个人免费额度最高
AI IDE 个人免费
免费（个人）· Pro $15/月

Codeium 旗下产品，最大卖点是补全无限量，个人用户永久免费。IDE 体验接近 Cursor，功能略简单，但对轻度使用者来说足够。适合：不想每月花钱、日常用 AI 辅助写代码但不依赖高强度 Agent 任务的开发者。

AWS Kiro
Amazon · 2026年初发布
AI IDE NEW 规范驱动
定价未公布

这个工具的思路和其他所有 IDE 都不同。它推行规格驱动开发（Spec-Driven Development）：你先描述需求，AI 生成需求文档（PRD）、设计文档（Technical Design）、再生成代码。全程留档，可追溯。

适合的场景：金融、医疗等受监管行业；需要代码审计、合规审查的团队；多人协作项目需要统一文档规范。如果你只是自己做项目，这套流程会觉得繁琐。

05

IDE 插件 / 平台助手（集成派）

不想换编辑器？这类工具以插件形式嵌入你现有的 IDE，哪里都能用，是目前企业采用率最高的形态。

GitHub Copilot
Microsoft / OpenAI · 最广泛使用
VS Code JetBrains Neovim 免费额度
免费 · Pro $10/月 · Business $19/月

装机量最大的 AI 编程工具，几乎支持所有主流编辑器。免费版已包含每月2000次补全+50次对话——入门性价比最高。现在支持多模型（GPT-5.4、Claude Sonnet 4.6、Gemini 3.1 Pro），不再绑定单一模型。

企业版有私有代码库过滤（不拿你的代码训练）、组织策略管理、审计日志等安全功能，这是很多大厂选择它的主要原因。

✦ 优势
  • 支持编辑器最多，哪都能用
  • 免费版已相当实用
  • 企业安全合规功能最成熟
  • 多模型选择
✦ 不足
  • 补全质量略逊 Cursor Supermaven
  • Agent 任务能力弱于专用 Agent 工具
Amazon Q Developer
Amazon · AWS 生态首选
VS Code JetBrains AWS 集成
免费 · Pro $19/月

如果你的工作大量涉及 AWS 服务（Lambda、DynamoDB、S3、CloudFormation 等），Q Developer 有独特优势：深度理解 AWS 架构，生成的代码天然符合 AWS 最佳实践，能自动生成 CDK/SAM 模板，直接在 IDE 里完成 AWS 资源操作。对非 AWS 用户，它没有显著优势。

Continue.dev
开源 · 高度可定制
开源 VS Code JetBrains 完全免费
免费（自付 API 费用）

开源插件，支持接入任意模型（Claude、GPT、DeepSeek、本地 Ollama 等），自己带 API Key。完全免费，高度可定制——可以自定义 Prompt、接入私有知识库、配置本地模型。适合：有隐私要求（代码不出局域网）或想精细控制成本的团队。

06

在线平台 / Playground

不安装任何东西，浏览器里直接用。适合：快速试验一个 API、在线跑代码原型、学生 / 非专业开发者。

Google AI Studio
Google · Gemini API 官方测试平台
免费额度高 多模态
免费 · API 按量计费

Gemini 系列模型的官方 Playground。免费额度在几大平台里最慷慨，支持图片/视频/音频/PDF 多模态输入，可直接在界面里调 Prompt、测试 API、导出代码。

定位很明确：这是给想用 Gemini API 做开发的人提供的调试工具，不是最终用户产品。如果你想在自己的应用里接入 AI，从这里开始实验效率最高。

Claude.ai / Anthropic Console
Anthropic
免费版可用
免费 · Pro $20/月

Claude 模型的官方入口。Claude.ai 是面向普通用户的对话界面，可以做代码问答、Code Review、架构讨论。Console（console.anthropic.com）是给开发者调试 API、做 Prompt 工程用的，可以精确控制参数、看 Token 用量。

Replit
Replit Inc. · 在线 IDE + AI
在线 IDE 免费入门
免费 · Core $25/月

浏览器内完整 IDE，可以运行代码、部署应用、协作开发，AI 嵌入编写流程。学生和教学场景最适合——零环境配置，打开浏览器就能跑代码。职业开发者通常会觉得功能有限，更适合做原型验证或个人小项目。

09

趋势与总结

这个领域迭代极快，每隔几个月格局就会变化。但有几个方向是确定的：

01
从"补全"到"工程决策"

2021年的 Copilot 只做代码补全。现在的顶级工具已经能独立理解需求、拆解任务、读懂整个代码库、做架构决策。AI 在工程流程里的参与深度还在持续加深。

02
Multi-Agent 并行成为下一个战场

单个 AI Agent 线性处理任务，已经不能满足复杂工程需求。Antigravity 的 Mission Control、Claude Code 的 Agent Teams，都在探索多个 Agent 并行分工的范式。这是未来1-2年的主要演化方向。

03
模型成本持续下降，工具竞争加剧

DeepSeek 的出现打破了"顶级能力必须高价"的假设。随着开源模型越来越强，工具层的差异化会越来越依赖工程质量和产品体验，而不是模型本身。

04
上下文长度成为关键竞争点

1M tokens 的上下文意味着 AI 可以一次性读入数十万行代码库，真正"理解"一个大型项目。这个能力正在从旗舰模型向普通模型普及，会根本性改变大型项目的 AI 辅助方式。

写在最后：这些工具的本质都是放大器——放大你已有的工程能力。一个对代码库、架构、业务逻辑有深刻理解的开发者，使用 AI 工具能做到事半功倍。如果基础不扎实，AI 可能只是把你引向错误方向的速度加快了而已。工具是杠杆，人才是支点。
